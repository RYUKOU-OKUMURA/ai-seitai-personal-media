# New Blog

このリポジトリのブログ記事を、既存トーンと frontmatter ルールに合わせて作成する。

## ワークフロー

### 1. 先に既存記事を確認する

- 必要に応じて `src/content/blog/` の既存記事を 1 から 2 本だけ読む。
- タイトルの付け方、`slug`、`excerpt`、締め方を既存記事に合わせる。

### 2. ファイル名と frontmatter を決める

- 保存先は `src/content/blog/` にする。
- ファイル名は日本語タイトルベースで作り、長いダッシュはハイフンに置き換える。
- `slug` は英小文字とハイフンだけで作る。
- 公開指示がない限り `draft: true` を使う。
- アイキャッチ未定なら `image: ""` にする。

```yaml
---
title: "メインテーマ——サブメッセージ"
slug: "english-slug-here"
publishedAt: "YYYY-MM-DDT00:00:00.000Z"
category: "カテゴリ名"
image: ""
excerpt: "要約文（1〜2文）"
draft: true
---
```

### 3. 本文を構成する

- 導入はテーマに合わせて書き分ける。毎回「結論から言うと」で始めない。
- `##` 見出しを 2 から 3 本入れる。
- 最後に `## 今日のまとめ` を置き、番号付きリストで要点を整理する。
- 締めは `でわでわ、次回の記事で！` にする。

## 文体ルール

- 一人称は「僕」にする。
- ですます調をベースにしつつ、口語を少し混ぜる。
- 読者の不安には寄り添うが、書き手は「すでに実践している側」の視点を保つ。
- 強調したい語だけを `**太字**` にする。
- `**` は日本語の括弧の内側に置く。

## 本文の型

1. 読者の課題や違和感をつかむ導入
2. 問題の正体を定義する H2
3. 解釈や実践方法を示す H2
4. 必要なら注意点や補足の H2
5. `## 今日のまとめ`

## 投稿ルール

- 投稿が必要なときだけ `npm run cms:post -- src/content/blog/<ファイル名>.md` を実行する。
- 実行前に `.env.local` に `MICROCMS_SERVICE_DOMAIN` と `MICROCMS_API_KEY` があるか確認する。
- microCMS 側のカテゴリ設定は手動前提なので、必要ならその旨を伝える。
